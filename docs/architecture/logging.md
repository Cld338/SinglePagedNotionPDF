# Logging & Monitoring

안정적인 백그라운드 작업 처리 및 추적을 위해 `winston` 기반의 중앙 집중형 로깅 시스템을 사용합니다.

## 1. 로거 설정 (`src/utils/logger.js`)
- **포맷**: 타임스탬프(`YYYY-MM-DD HH:mm:ss`)와 로그 레벨을 포함하여 출력합니다.
- **출력 방식**: 현재 Console Transport를 사용하여 색상이 적용된 로그를 표준 출력(STDOUT)으로 내보냅니다. 이는 Docker 컨테이너 환경에서 `docker logs` 명령어를 통해 로그를 수집하기 가장 적합한 형태입니다.

## 2. 작업 이력 추적
- 큐에 작업이 추가될 때, Worker가 처리를 시작할 때, 그리고 완료되거나 실패할 때 모든 단계에서 고유 식별자(`job.id`)와 대상 URL을 포함하여 로그를 남깁니다.
- 치명적인 에러(브라우저 크래시 등) 발생 시 `logger.error`를 통해 스택 트레이스를 보존하여 문제 해결을 돕습니다.
