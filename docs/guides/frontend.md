# Frontend Client Logic

프론트엔드는 별도의 프레임워크 없이 순수 HTML/JS/CSS로 작성되었으며(`public/index.html`), 비동기 작업 큐의 상태를 사용자에게 매끄럽게 보여주기 위한 로직이 포함되어 있습니다.

## 1. 작업 큐 상태 폴링 (Polling)
서버가 202 Accepted와 함께 `jobId`를 반환하면, 클라이언트는 해당 Job이 완료될 때까지 상태를 지속적으로 확인합니다.
- **주기**: `setInterval`을 활용하여 3초(3000ms)마다 `GET /job-status/:id` 엔드포인트를 호출합니다.
- **성공 처리**: 상태가 `completed`로 변경되면 폴링을 중단하고 반환된 `downloadUrl`을 통해 보이지 않는 `<a>` 태그를 생성하여 파일 다운로드를 자동 트리거합니다.
- **실패 처리**: 상태가 `failed`일 경우 폴링을 중단하고 에러 메시지를 `alert`으로 노출합니다.

## 2. UI/UX 상태 관리
PDF 변환 작업(최대 2분 소요) 동안 사용자의 중복 클릭을 방지하고 작업 진행 상황을 알립니다.
- **제출 중 상태**: 'Convert to PDF' 버튼을 `disabled` 처리하고 텍스트를 변경합니다.
- **로딩 오버레이**: 화면 전체에 `.loading-overlay` 요소와 스피너를 표시하여 대기열 등록 및 변환 진행 상태를 시각적으로 안내합니다.
